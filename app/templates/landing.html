Judging Begins

Requirements

Your social media website must be compliant on certain fronts for testing. These will hopefully give you valuable insight into full-stack web development and general software engineering practices!

Responsive Design

Your website must be usable on mobile, tablet and desktop environments. You're allowed to use any framework or library out there so don't get too caught up on this.

Admin APIs

You must provide an admin API endpoint for creating and managing users and posts. These will be used for benchmarking and testing your website! Read the docs for more information on what to do.

High Performance

On the frontend, page load times will be used as a metric for judging. Make sure your backend and frontend are well designed and cohesive enough to provide a smooth UX.

Secure

You must provide secure interfaces to all endpoints that need to be secured. No storing unhashed passwords or RCEs are allowed!

Community

You must have support for posts, comments, followers and a customized feed for each user.

Support

You must have a way to manage users, posts, and the general platform via an admin dashboard. It doesn't have to be special, it just has to exist.

Scalable Infrastructure

Another key thing we're looking for is the ability to scale well. Make sure you're being performant with your queries and web server configuration.

Intuitive UI/UX

Be creative! Bonus points for going into a special style or through interactive animations that're tied into your web application.

Social Media Problem Case

In doing the Social Media Problem Case, you're going to gain the confidence to be able to build whatever you want on the web. Good luck and have fun!
